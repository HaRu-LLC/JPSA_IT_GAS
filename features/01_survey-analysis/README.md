## セットアップ手順

### Step 1: Google Formsのスプレッドシートを開く

Google Formsの回答が自動連携されているスプレッドシートを開きます。シート名が「フォームの回答 1」であることを確認してください。

※ シート名が異なる場合は、`01_Main.gs` の `CONFIG.SHEETS.RAW` を実際のシート名に変更してください。

### Step 2: Apps Scriptエディタを開く

メニューバー → 「拡張機能」 → 「Apps Script」

### Step 3: GASファイルを追加

Apps Scriptエディタで、以下の手順で8つのファイルを追加します。

1. 左側の「ファイル」パネルの「+」ボタン → 「スクリプト」を選択
2. ファイル名を入力（`.gs` は不要。例: `01_Main`）
3. 対応する `.gs` ファイルの内容をコピー＆ペースト
4. 8ファイルすべて繰り返す

**重要**: `01_Main` → `02_DataProcessor` → ... の順に追加してください。

### Step 4: 保存して実行

1. 「Ctrl+S」で保存
2. スプレッドシートに戻る（タブを切り替え）
3. ページをリロード（F5）
4. メニューバーに「🔧 IT部会分析」が表示されることを確認

### Step 5: 初回実行

1. メニュー「🔧 IT部会分析」→「📊 全データを再処理する」をクリック
2. 初回は権限の承認ダイアログが表示されます → 「許可」をクリック
3. 処理完了のダイアログが表示されるまで待機

### Step 6: WebAppダッシュボードのデプロイ（推奨）

1. Apps Scriptエディタ → 右上の「デプロイ」→「新しいデプロイ」
2. 「種類の選択」→ 歯車アイコン → 「ウェブアプリ」を選択
3. 以下を設定:
   - **説明**: 「JPSA IT部会 アンケート分析ダッシュボード」
   - **次のユーザーとして実行**: 「自分」
   - **アクセスできるユーザー**: 「組織内の全員」（または「リンクを知っている全員」）
4. 「デプロイ」をクリック
5. 表示されたURLが**ダッシュボードのアクセスURL**です
6. このURLを運営メンバーに共有すれば、誰でもブラウザでダッシュボードにアクセスできます

**注意**: コードを更新した場合は「デプロイを管理」→「新しいバージョン」でデプロイを更新してください。

### Step 7: Looker Studio連携（オプション）

1. メニュー「🔧 IT部会分析」→「🔗 Looker Studio連携URLを生成」をクリック
2. 表示されたURLをクリックしてLooker Studioを開く
3. データソースの接続を承認
4. `Looker_Studio構築ガイド.md` を参照してダッシュボードを構築

## 指定月 特別分析について

メニュー「🔍 指定月の特別分析」を実行すると、月選択ダイアログが表示されます。分析したい月を選ぶと、以下を含む包括的なレポートが「指定月分析」シートに生成されます。

- **全体比較KPI**: 指定月 vs 全体平均（達成度、登録率、リピーター率を比較）
- **職業分布**: その月の参加者プロファイル
- **認知経路分布**: どこから来たか
- **参加目的カテゴリ**: 何を求めて来たか
- **達成度分布**: 満足度の内訳
- **月固有の質問**: その月にしかない質問項目の集計（従業員数、AI関連サービス等）
- **頻出ワード**: その月のテキストから抽出したワードランキング
- **原文一覧**: 参加目的・実行アクション・感想のテキスト全文

Looker Studioでは、このシートを5ページ目のデータソースとして接続し、分析月を変更するたびにGASメニューから再実行→Looker Studioが自動反映されます。

## 毎月の運用フロー

1. Google Formsで新しい勉強会のアンケートを収集
2. 回答が「フォームの回答 1」シートに自動蓄積
3. メニュー「➕ 新しい月のデータを追加処理」を実行
4. 必要に応じて「🔍 指定月の特別分析」で深掘り
5. メニュー「🖥️ レポートページを開く」でHTMLレポートを確認
6. Looker Studioダッシュボードは自動的に最新データを反映

※ 自動更新トリガーを設定済みの場合、Step 3は毎週月曜に自動実行されます。

## トラブルシューティング

**Q: メニューが表示されない**
→ ページをリロードしてください。Apps Script保存後、onOpen()は次回シート読み込み時に実行されます。

**Q: 「権限が必要です」と表示される**
→ 初回実行時は正常です。「許可」→ アカウント選択 → 「詳細」→「（プロジェクト名）に移動」→「許可」の順に進んでください。

**Q: カラムが見つからないエラー**
→ Google Formsの質問文が変更された可能性があります。`02_DataProcessor.gs` の `COLUMN_MAP` を実際の質問文に合わせて修正してください。

**Q: 指定月分析で月が表示されない**
→ 先に「全データを再処理する」を実行してください。処理済みデータがないと月の一覧を取得できません。
