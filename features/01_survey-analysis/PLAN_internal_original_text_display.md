# 内部運営向け原文表示 実装プラン

## 目的

お客様の声・原文一覧・月別詳細では、内部運営チームが文脈を判断できるよう、回答本文中の個人名を `[氏名非表示]` に置換せず原文表示する。一方で、メールアドレス・電話番号・個人情報列そのもの・Gemini外部送信は保護する。

## 現状の問題

- `09_WebApp.gs` の `getFullData()` が画面表示用データを返す時点で、本文セルに `maskDashboardPublicText_()` を適用している。
- `getMonthDetailData()` も月別詳細テキストや `topWords` 生成前の本文に同じマスクを適用している。
- `Dashboard.html` の `escPublicText_()` が、HTMLエスケープだけでなく `redactPersonalInfo_()` による氏名マスクも行っている。
- そのため、画面表示用データから実名が失われ、ブラウザ側では元に戻せない。

## 実装方針

### 1. サーバー側データ返却を「内部表示用」に変更

- `getFullData()` は、`氏名` / `メールアドレス` / `電話番号` / `参加者キー` / `紹介者名` の列を引き続き除外する。
- `getFullData()` の本文セルには、氏名マスクを適用しない。
- ただし本文中のメールアドレス・電話番号は引き続きマスクする。
- 参加者識別は既存どおり匿名化 `参加者ID` を返す。

### 2. 月別詳細の原文表示を実名表示に変更

- `getMonthDetailData()` の `getTexts()` と `specialData` は、氏名マスクを適用せず本文を返す。
- 本文中のメールアドレス・電話番号はマスクする。
- `topWords` は本文表示とは別に、マスク表記や必要に応じた人名ノイズを除外して集計する。

### 3. 画面表示ヘルパーを整理

- `Dashboard.html` の `escPublicText_()` は、HTMLエスケープを主責務にする。
- 氏名列由来の置換処理は表示経路から外す。
- 本文中のメールアドレス・電話番号だけは表示前にもマスクできるよう、別ヘルパーに分離する。

### 4. Gemini送信前マスクを維持

- `refreshGeminiVocSummary()` から渡すテキストは、画面表示用の原文を元にしてよい。
- `generateGeminiVocSummary()` では、送信直前に `氏名` 列由来の名前パターン・メール・電話番号を必ずマスクする。
- Gemini送信前マスクは、内部表示仕様とは別の外部送信保護として残す。

### 5. ワード集計の扱いを明確化

- `[氏名非表示]` / `[メール非表示]` / `[電話番号非表示]` は、既存データや過去処理結果に残る可能性があるため、引き続き集計前に除外する。
- 実名表示に戻した後、人名をワード集計に含めるかは運営判断に依存する。初期実装では、原文表示を優先し、ワード集計からの人名除外は追加要件として扱う。

## 実装ステップ

1. `09_WebApp.gs` に表示用マスクと外部送信用マスクを分離する。
   - 表示用: メール・電話番号のみマスク
   - 外部送信用: 氏名・メール・電話番号をマスク

2. `getFullData()` を修正する。
   - PII列除外は維持
   - 本文中の氏名置換を停止
   - メール・電話番号マスクのみ適用

3. `getMonthDetailData()` を修正する。
   - 原文返却は氏名置換なし
   - メール・電話番号マスクのみ適用
   - `topWords` の既存マスク表記除外は維持

4. `Dashboard.html` を修正する。
   - `escPublicText_()` をHTMLエスケープ中心に変更
   - 表示時の氏名列ベース置換を停止
   - Gemini要約へ渡す前のクライアント側マスクは補助扱いにし、最終防衛線はサーバー側に置く

5. 確認する。
   - 原文一覧に実名が表示される
   - メールアドレス・電話番号は表示されない
   - PII列はダッシュボードデータに含まれない
   - Gemini送信前には氏名・メール・電話番号がマスクされる
   - `[氏名非表示]` 由来の語がワードクラウドに出ない

6. 反映する。
   - 構文確認
   - `clasp push --force`
   - デプロイ更新はユーザーが実施

## 残す保護

- 個人情報列そのものは返さない。
- 参加者の継続識別は匿名化IDで行う。
- 外部AI送信時は氏名・メール・電話番号をマスクする。
- HTML表示では必ずエスケープする。
