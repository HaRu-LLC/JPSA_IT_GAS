# レポートページ完了条件チェックリスト

## A. データ連携の成立（必須）
- [ ] WebアプリURLを開くと、関数一覧ページではなくダッシュボードUIが表示される
- [ ] 初期表示で `データの読み込みに失敗` が出ない
- [ ] `開催月` / `職業` / `認知経路` の各フィルタで「全て」以外の候補が1件以上出る

## B. 月表示仕様（必須）
- [ ] 月関連ラベルは `YYYY年M月` 文字列で表示される（例: `2026年1月`）
- [ ] `2026-02-01 00:00:00` のような日時表示がフィルタ/横軸/指定月分析に残っていない
- [ ] 概要タブの月別グラフ横軸が月の時系列順で並ぶ
- [ ] 指定月特別分析の月候補がドロップダウンで選択できる

## C. 参加者分析（必須）
- [ ] フィルタに `受講状況` が追加されている
- [ ] `受講状況=未受講以外` を選ぶと、未受講データが除外される
- [ ] 参加者分析タブに `受講状況` グラフが表示される
- [ ] 指定月特別分析に `受講歴（未受講/受講済み/JPSA会員）` グラフが表示される

## D. Voice of Customer（必須）
- [ ] ワードクラウドとTop20グラフが表示される
- [ ] 原文一覧の前に「要約・傾向」セクションが表示される
- [ ] 要約に回答数/達成度/主傾向（参加目的・アクション・認知経路）が出る

## E. 異常時の挙動（必須）
- [ ] データが空の場合、処理実行手順の案内が表示される（TypeErrorで落ちない）
- [ ] データ健全性に問題がある場合、警告通知が画面上部に表示される

## F. デプロイ反映（必須）
- [ ] 修正後コードを `clasp push --force` で反映済み
- [ ] Webアプリを「ウェブアプリ」種別で再デプロイ済み
- [ ] 再デプロイURLでA〜Eを満たす
