# AGENT.md

このファイルは、`JPSA_IT_GAS` リポジトリ全体を扱うエージェント向けの作業ガイドです。

## 1. リポジトリの位置づけ
- このリポジトリは、JPSA IT部会として複数機能を束ねる全体整合用の親リポジトリ。
- 機能固有のコード・運用資料・作業メモは `features/` 配下に置く。
- 複数機能にまたがる方針、共通ルール、全体READMEはルートに置く。

## 2. 現在の機能構成
- `features/01_survey-analysis`: 既存の「勉強会アンケート分析」機能一式。
- 将来の新規機能は `features/02_*` 以降で追加する。

## 3. 変更時の基本ルール
- 機能固有の変更は、対象機能ディレクトリの中で閉じる。
- ルートに新規ファイルを置くのは、全体方針・共通資産・横断ドキュメントに限る。
- 各機能に個別運用ルールがある場合は、その機能ディレクトリ直下の `AGENT.md` を優先して参照する。

## 4. 今回時点の全体ファイル
- `README.md`: リポジトリ全体の案内。
- `AGENT.md`: この全体ガイド。

## 5. 新機能追加の原則
1. `features/02_<feature-name>` のようなディレクトリを作る。
2. 機能のコード・設定・README・運用資料はその配下に置く。
3. 全体に影響するルールだけをルートの文書へ追記する。
